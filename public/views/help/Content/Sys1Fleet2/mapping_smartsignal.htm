Mapping System 1 and SmartSignal Machines.

The Mapping feature allows administrators to correlate data from identical assets that are being monitored simultaneously by System 1 and SmartSignal. System 1 and SmartSignal use different naming conventions to identify the same assets.

The following figure shows the hierarchy of the System 1 and SmartSignal data sources. The highlighted sections in the drawing show three assets that are each being monitored at the same time by System 1 and SmartSignal.

As shown in the figure, Machine 2 in the System 1 hierarchy is the same piece of equipment as Asset X in the SmartSignal hierarchy. In this procedure is a turbine.

The purpose of mapping equipment that is being monitored by SmartSignal and System 1 is to consolidate and present both sets of alarm and event data collected by each system on the Dashboard and Alarm pages of the System 1 Fleet Management portal.

  After you complete mapping one or more pieces of equipment to each other, the System 1 Fleet Management portal displays an icon next to the asset's name in the portal on both the Dashboard and Alarm pages.

Before you begin

Identify the identical assets that are being monitored by both SmartSignal and System 1 Fleet Management. Each application uses a unique naming convention for its components. Make a list of the machines that you want to map.

For example, in System 1, you have a wind turbine identified as Power Turbine[Platform_F_1ac60089-5479-4b04-aa6d-39bf451779f1~154]

 

The same wind turbine in SmartSignal is identified as Test Wind Turbine 1[SS-175_5d5dd94e-a5d4-4cca-a019-e9ab38f25fc0~0f1a36fa-4a87-4cbc-896d-ed01ddd7e956]

Restrictions:

  • You cannot map an asset to a folder.
  • You can only map a SmartSignal Asset to a System 1 Asset (Train or Machine)
  • The mapping ratio is 1:1.

Exporting Hierarchies

Follow the instructions in this section to export all hierarchy members from SmartSignal and System 1 to a single spreadsheet file in the .xls format. You must open the file and perform the mapping tasks in a spreadsheet that can interpret an .xls file, for example, Microsoft Excel.

  1. Complete your mapping plan.
  2. From the home page, click Administration. The Users page is displayed.
  3. Click Mapping. The Mapping page is displayed as shown below. If the Mapping page displays No Data Available, then no previous mappings have been performed. The message does not indicate that there is no data to be mapped.


  4. From the drop-down menu, select a Fleet name, for example, GE.
  5. In the Default Asset View drop-down, select System 1 or SmartSignal. The selection you make controls the contents of the left column of the spreadsheet.
  6. A file named S1Fleet-Customer(xx)-enterprise-mapping.xls is created and stored in the Downloads folder. A button appears in the bottom left corner of the browser window.
  7. Click the button to open the exported spreadsheet.
  8. Click Enable Editing.
  9. Click and drag the column borders to create three distinct columns. Make sure that the empty "B" column named SmartSignal Mappings is in the middle of the spreadsheet.

Map the assets

Important: Do not cut and paste asset names from the "A" or "C" columns when you populate the "B" column. Copy and paste the asset names, instead.

In this procedure, you map two assets (see the following table).

SmartSignal Test Wind Turbine Group[SS-175_5d5dd94e-a5d4-4cca-a019-e9ab38f25fc0~d506a0bd-ecc3-40b2-b062-23c9d0530643] (see highlighted row C5 in the figure below)

System 1 Power Turbine[Platform_F_1ac60089-5479-4b04-aa6d-39bf451779f1~154] (see highlighted row A8 in the figure below.)

 

  1. In the left column, identify the System 1 asset that you want to map to its SmartSignal counterpart (see A8 in the figure above).
  2. In the right column, select the SmartSignal asset you want to map to the System 1 asset selected in step 1 (see C5 in the figure above).
  3. Copy the asset in the right column.
  4. Click in column B8.
  5. Click Paste.
  6. Click Save to update the spreadsheet you exported and preserve the mapping information.
  7. Verify that the updated spreadsheet looks similar to the following figure.

Option: Rename the mapped spreadsheet.

Import the Mapped Spreadsheet File

  1. Bring up the System 1 Fleet Management Mapping page.
  2. In the right pane, click Choose File.
  3. Navigate to the file that was mapped in the previous procedure.
  4. Click once on the name of the mapped file.
  5. Click Open. The Mapping page is displayed.
  6. In the right pane of the Mapping page, click Import File.
  7. The bottom pane of the Mapping page should resemble the following figure.

Verifying the Mapping on the Dashboard page

  1. Click Dashboard.
  2. Click the fleet name that contains the systems you mapped, for example, GE.
  3. Click System 1 Hierarchy.
  4. Drill down in the hierarchy menu to the name of the Power Turbine asset that you mapped to the SmartSignal Test Wind Turbine.
  5. Verify that the System 1 Power Turbine asset is marked with the mapped icon.
  6. Click SmartSignal Hierarchy.
  7. Drill down in the hierarchy to the name of the SmartSignal Test Wind Turbine that you mapped to the System 1 Power Turbine.
  8. Verify that the System 1 Power Turbine asset is marked with the mapped icon.
