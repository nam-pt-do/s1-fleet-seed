Selecting Columns for the Alarm Table

The information in the Alarms List table depends on the columns chosen for display.

To select one or more columns, complete the following steps;

  1. From the Dashboard page, click Alarms. The Alarms page is displayed.
  2. Click the Show/Hide Columns icon . A list of the column titles that can be shown or hidden in the Alarms List table is displayed (see the following figure).


  3. Click the check box to the left of a column name to display the column in the Alarms List table.
  4. Clear the check box to the left of a column name to hide the column from the Alarms List table.

If you add columns to the Alarms List table, the table expands horizontally. A scroll bar might appear at the bottom of the table. In some cases, you might need to use the horizontal scroll bar to view all columns in the table. The following table defines the column names.

This Column Indicates This information
Last Occurred The GMT date and time when this event was last recorded.
First Occurred The GMT date and time when this event was first recorded.
Data Source The source system where the event occurred. The source can be System 1 or SmartSignal.
Fleet Level

Fleet normalized severity level for this event. Severity levels are Low, Medium or High.
Source Level Original source system severity level for this event.
Asset Path Displays the path for the asset from the asset hierarchy. Only the first two levels of the asset path are displayed. Hover over the column to display the full path, for example, ../San Ramon Power/Compression.
Asset

The name of the asset on which this event was occurred.

  • If the alarm is from a SmartSignal asset, then the <asset name> is displayed.
  • If the alarm is from a System 1 Train, then the <train name> is displayed.
  • If the alarm is from a System 1 Machine , then the <machine name> is displayed, for example, LM2500.
Point/Variable

The name of the System 1 point/System 1 variable, or the SmartSignal model tag creating this event.

  • If the alarm is from a SmartSignal diagnostic rule, then <null> is displayed.
  • If the alarm is from a SmartSignal tag rule, then <model_tag> is displayed.
  • If the alarm is from a System 1 point, then <point name> is displayed.
  • If the alarm is from a System 1 variable under a point, then <point name> + "/" + <variable name> is displayed.
Type/Message For System 1, the column displays the alarm type, for example, Software alarm A. For SmartSignal, the column displays the message of the advisory. If the alarm is from System 1, the display is <type = description from Fleet database event table>. If the alarm is from SmartSignal, then the display is <message = description from the Fleet database event table>.
Category The category of the event (Asset Protection, Asset Management, Instrument Protection, Instrument Management]). For SmartSignal in this release, the category is always blank.
Count The number of events that have fired for this asset for this event type from this point/variable for the last year.
Density (%) For SmartSignal, density is the number of advisory events divided by the number of observations since the first advisory event firing. For System 1 alarms, the Density column is always blank.
Active Displays the alarm condition status of the alarm. The status is either YES or NO.
Acknowledged Status of the acknowledgment action. The status is either <YES> or <NO>.
Time of Acknowledgment Last Date and Time when this event was acknowledged. Depending on user preference settings, time can be local time or GMT.
Event ID Original source system event identification.
Source ID Original source system identification of the point, variable, or tag that generated this alarm.
