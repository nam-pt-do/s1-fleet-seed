Changing the Index Threshold Values

System 1 Fleet uses index threshold values to color-code the health status of enterprises, folders and nodes in Health Index View. Customers can configure different threshold values for each enterprise.

To change the index threshold values,complete the following steps;

  1. Launch the pgadmin3 client tool under C:\Program Files (x86)\Fleet2.0\fleetdb\pgsql\bin\ pgAdmin3.exe.
  2. Connect to the System 1 Fleet Management database. To log in, enter the User ID and Password assigned during database installation.
  3. Execute the following queries with appropriate values. Replace <EnterpriseId> in the query with the actual enterprise identifier.
  Perform severity mapping changes immediately after installing System 1 Fleet Management. After events are synced, changing severity mapping might result in an erroneous display.

update fms.severity_mapping set normalized_severity = 1 where source_system = 'SYSTEM1' and source_system_severity = 1;

update fms.severity_mapping set normalized_severity = 2 where source_system = 'SYSTEM1' and source_system_severity = 2;

update fms.severity_mapping set normalized_severity = 2 where source_system = 'SYSTEM1' and source_system_severity = 3;

update fms.severity_mapping set normalized_severity = 3 where source_system = 'SYSTEM1' and source_system_severity = 4;

update fms.severity_mapping set normalized_severity = 3 where source_system = 'SMARTSIGNAL' and source_system_severity = 1;

update fms.severity_mapping set normalized_severity = 2 where source_system = 'SMARTSIGNAL' and source_system_severity = 2;

update fms.severity_mapping set normalized_severity = 2 where source_system = 'SMARTSIGNAL' and source_system_severity = 3;

update fms.severity_mapping set normalized_severity = 1 where source_system = 'SMARTSIGNAL' and source_system_severity = 4;

update fms.severity_mapping set normalized_severity = 1 where source_system = 'SMARTSIGNAL' and source_system_severity = 5;
