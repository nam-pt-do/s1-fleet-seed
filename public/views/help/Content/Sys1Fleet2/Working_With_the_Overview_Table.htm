Working With the Grid View Table

After you log in to System 1 Fleet Management, the Dashboard is displayed a table that contains an overview of Fleet health parameters. This section explains how to use the icons and controls in the grid view to gather data about the Fleet sites, trains, train-machines and machines (see the following figure).

Selecting a Customer

The Customer drop-down menu displays only the customers that were assigned to you when your account was created.

  1. Click the down arrow in the Customer drop-down menu (see the following figure).
  2. Select a customer name. The Grid View table is updated to show Status, Alarm and related data for sites associated with the selected customer.

Using the Hierarchy to Navigate the Fleet Sites

Use the Hierarchy menu to select a site and drill down through folders, machine-trains and machines associated with the selected customer. The following table describes the icons that designate layers of the Hierarchy.

Understanding the Status column icons

 

Using the Links in the Name Column to Navigate the Fleet Hierarchy

  1. In the Name column, click the name of the Site or other Fleet component.
  2. An Open Folder pop-up is displayed.
  3. The Grid View is updated with the contents of the folder.
  4. Continue to click the Name in the Name column until you have reached the site, folder, train, train-machine or machine you want to view.

The following table describes the columns in the Overview index table.

Column Description
Name The name of the selected customer's Enterprise, Train, Machine, or Asset.
Status Indicates whether the machine, train or asset is running.

SS Shows the SmartSignal health index percentage.

Understanding the Icon in the Status column for a Site or Folder

The status column can display a red, green, or gray bar-shaped icon. The information provided by the presence and color the bar is different, depending on the selection you made when navigating the hiearchy, or drilling-down into the Fleeet using the links in the Name column.
