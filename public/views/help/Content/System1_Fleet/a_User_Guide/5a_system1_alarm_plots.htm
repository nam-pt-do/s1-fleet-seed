System 1 Alarm Plots

System 1 alarms are triggered when the installed sensors indicate when rotating-equipment fitness deviates from defined ranges, triggering alarm severities ranging from 1 (lowest) to 4 (highest).

When you navigate to the Alarms page and select a System 1 alarm from the list, you can click the Alarm Plot icon to generate default plots illustrating alarm data.

The top chart, which is shown below, illustrates the trends over a defined interval. Data is represented by the black line and shows where the alarm was triggered. The chart also contains a black vertical line representing the last time the alarm occurred on the timeline, with a 75/25 illustrated division so you can see the data points going back for 75% of the entire timeline, and 25% forward.

If setpoints have been configured for the point, they also appear in the plot. For information, see Setpoints.

The initial chart displays the default time span, but you can replot by selecting a different interval from the drop-down list on the right.

For alarms on points with high resolution dynamic data available, a waveform chart appears below the trend chart, which illustrates displacement distances in different formats. Vibration data can be represented in distance, velocity, or acceleration on the x-axis, and displacement measurements (for example, micrometers) on the y-axis.

If data is not available for one or both plots, the application displays a NO DATA AVAILABLE or NO WAVEFORM DATA AVAILABLE message.

You can change the plot parameters in either chart using the drop-down lists.

Selection Description
Mode

Synchronous vibration is typically used for collecting vibration data correlated to running speed using a keyphasor.

Asynchronous Asynchronous data is typically used for machines with higher order vibration frequencies or machines without a speed reference; For example, gearboxes or rolling element bearings where a broader frequency span and higher resolution spectrums are required.
